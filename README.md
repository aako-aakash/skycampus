## вңҰ What is SkyCampus?

SkyCampus is a full-scale university social ecosystem where students can build academic profiles, connect with peers, create and interact with posts, join interest-based communities, chat in real time, and receive intelligent AI-powered recommendations вҖ” all in one platform.

This is not a tutorial project. It is engineered like a production startup product вҖ” with a clean microservice architecture, real-time infrastructure, vector-based AI matching, and cloud-native deployment across Vercel, Railway, and Render.

## вңҰ Security Architecture

<br />

```
вң“  bcrypt password hashing          12 salt rounds вҖ” brute force resistant
вң“  JWT access tokens                15 min expiry вҖ” minimal exposure window
вң“  Refresh token rotation           7 day tokens, rotated on every use
вң“  Token reuse detection            Old refresh tokens invalidated immediately
вң“  HTTP-only secure cookies         JWT inaccessible to JavaScript вҖ” XSS proof
вң“  Helmet.js                        14 security response headers on every request
вң“  CORS whitelist                   Only CLIENT_URL origin accepted
вң“  Zod input validation             All request bodies sanitized before DB
вң“  Prisma parameterized queries     SQL injection impossible by design
вң“  Rate limiting                    10 req/15min auth  вҖў  200 req/15min global
вң“  Resource ownership checks        Every mutation verifies req.userId === owner
```

## вңҰ Engineering Highlights

<br />

**Cursor-based pagination** вҖ” uses the last item ID as a stable anchor instead of page numbers, preventing duplicates in infinite scroll even as new content appears in real time.

**Optimistic UI** вҖ” TanStack Query's `onMutate` updates the feed instantly before the server responds. If the request fails, it automatically rolls back. Likes feel instant.

**Repository pattern** вҖ” raw DB access isolated in repository files. Controllers stay thin, services stay pure, repositories are mockable. Swapping Prisma for raw SQL requires touching zero business logic.

**Atomic like counters** вҖ” `likesCount` is incremented in the same `$transaction` as the like record insert. Impossible to get out of sync under concurrent requests.

**Graceful Redis degradation** вҖ” every cache operation wrapped in try/catch. Redis down = silent fallback to PostgreSQL. Zero downtime, zero user impact.

**CPU-only AI inference** вҖ” all-MiniLM-L6-v2 produces 384-dim embeddings in ~50ms on CPU. No GPU required. AI features run on any free-tier cloud instance.
